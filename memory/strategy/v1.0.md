# GAN Evolution Engine v2.0 - 初始策略

**版本**: v1.0  
**创建时间**: 2026-03-26  
**状态**: 初始策略

---

## 📐 核心原则

### 1. 代码质量

- 所有代码必须编译通过
- 所有测试必须通过
- ESLint 检查必须通过
- 无 Critical/High 安全问题

### 2. 安全要求

- 所有外部输入必须验证
- 所有函数必须有错误处理
- 敏感数据绝不日志输出
- 密码必须 bcrypt 加密
- JWT 必须设置过期时间

### 3. 性能要求

- API 响应时间 < 200ms (p95)
- 数据库查询必须有索引
- 避免 N+1 查询
- 缓存频繁访问的数据

### 4. 测试要求

- 单元测试覆盖率 > 80%
- 关键路径必须有集成测试
- 边界条件必须测试

### 5. 文档要求

- 所有公共 API 必须有文档
- 复杂逻辑必须有注释
- 策略变更必须记录

## 📊 问题模式库

### P001: 敏感数据日志暴露

**描述**: 密码/token 等敏感数据被记录到日志  
**修复**: 使用日志脱敏，敏感字段显示为 [REDACTED]  
**场景**: 所有日志输出

### P002: Token 状态管理缺失

**描述**: JWT token 未存储到数据库，无法撤销  
**修复**: 存储 token 到数据库，支持撤销机制  
**场景**: 所有认证系统

### P003: 邮箱验证流程缺失

**描述**: 用户注册后未验证邮箱即可登录  
**修复**: 添加邮箱验证流程，验证后才能登录  
**场景**: 所有用户系统

### P004: 速率限制不足

**描述**: 认证路由速率限制过松  
**修复**: 认证路由≤5 次/15 分钟，只计失败请求  
**场景**: 所有公开 API

### P005: Token 过期时间过长

**描述**: 重置 token 过期时间超过 15 分钟  
**修复**: 重置 token ≤15 分钟  
**场景**: 所有临时 token

### P006: Token 撤销机制缺失

**描述**: JWT token 一旦签发，过期前始终有效  
**修复**: 使用 token 版本号或黑名单机制  
**场景**: 所有 JWT 系统

### P007: 密码策略不足

**描述**: 密码未要求特殊字符  
**修复**: 密码必须包含特殊字符，最小长度 10  
**场景**: 所有用户系统

## 🔄 进化流程

### Round 1
1. Generator 实现基础功能
2. Discriminator 审查 (安全/性能/可读性)
3. Evolver 更新策略到 v1.1
4. 识别新问题模式

### Round 2
1. Generator 修复 Critical/High 问题
2. Discriminator 再审
3. Evolver 更新策略到 v1.2
4. 更新模式库

### Round 3+
- 重复直到质量达标
- Critical/High = 0
- 测试覆盖 > 80%
